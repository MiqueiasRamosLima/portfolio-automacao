Fluxo de integração Make.com + Kommo
Como funciona na prática
1
Webhook recebe evento
Formulário ou lead qualificado dispara POST para o webhook do Make.com com todos os dados do contato.
POST https://hook.make.com/seu-webhook
2
Make.com prepara a requisição
Módulo HTTP monta o body da API do Kommo com os campos mapeados: nome, empresa, valor, pipeline_id e tags.
HTTP Module › Make a Request
3
Autenticação OAuth 2.0
Token de acesso do Kommo é obtido via OAuth 2.0 e renovado automaticamente quando expira — tudo gerenciado pelo Make.com.
Authorization: Bearer {access_token}
4
Criar lead via API REST
POST para /api/v4/leads com o JSON do lead. Kommo retorna ID e dados do lead criado, incluindo link direto para o card no CRM.
POST https://conta.kommo.com/api/v4/leads
5
Salvar referência no Supabase
ID retornado pelo Kommo é salvo na tabela leads_qualificados para manter rastreabilidade entre os dois sistemas.
UPDATE leads SET kommo_id = {{id}} WHERE id = {{uuid}}
6
Webhook reverso (Kommo → Make)
Quando SDR muda o status do lead no Kommo, um webhook dispara automaticamente para o Make.com atualizar o Supabase e notificar o time.
Kommo Webhooks › lead.status_changed
